ZDP Demo Application

This sample application demonstrates using ZDP requests to reveal properties of remote devices.

Up to 8 nodes may participate. Each node is identified by a unique short address specified as the CS_NWK_ADDR parameter in the configuration.h file. This implies that each node shall be programmed with a separately compiled firmware image. The device type is set at run time by the application depending on the short address value. Namely, if the short address equals 0x0000, then the application decides that this is the coordinator, otherwise, the device type is set to router.

On startup, the nodes form a network (a successful join to the network is indicated by the green LED flashing).

Connect one node serially to a PC to test the applicaiton functionality with Windows HyperTerminal. After you set proper connection settings, you will be prompted with the following menu:
