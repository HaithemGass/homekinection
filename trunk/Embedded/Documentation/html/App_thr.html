Throughput Test Application

This application allows measurement of maximum application data throughput in various transmission scenarios, such as with/without encryption, with/without APS level acknowledgments, with different payload sizes, etc. One coordinator and one router nodes are participating.

The device type is specified in the configuration.h file by the APP_DEVICE_TYPE parameter. Therefore firmware images for the coordinator and the router shall be compiled separately.

The network start procedure is initiated automatically on device startup.

Upon network join the router node after TIMER_DELAY_PERIOD starts to continuously send data packets over the air to the coordinator. The coordinator receives data frames and measures the application level throughput based on the number of payload bits received from the air and forwards this value to a PC via USART once every TIMER_TRACK_PERIOD ms. Thus measured values can be observed in a terminal window (for example, Windows HyperTerminal). See COM port connection settings.

In order to measure the maximum data throughput with encrypted data frames, the application shall be compiled with properly configured security-related parameters.

LED Indications

If LEDs are present on the board, then they are used by the application:

  • The green LED is periodically blinking with 500ms interval when device is joining the network and is ON if it is in the network.
  • The red LED indicates packet transmission is indicated on the router.
  • The yellow LED is flashing when the coordinator is receiving data.
  • The red LED is flashing when the coordinator is forwarding measureds value to the USART interface.

Important configuration parameters

APP_DEVICE_TYPESpecifies the device type
APP_USE_APS_ACKSpecifies whether or not the APS-level acknowledgement (end-to-end) shall be requested for transmitted data frame.
TIMER_DELAY_PERIODThe time interval (ms) after the network join when the coordinator starts measuring throughput and the router starts sending frames.
TIMER_TRACK_PERIODThe time interval (ms) used on the coordinator for throughput measurement and output to the serial interface.
CS_CHANNEL_MASKSpecifies a channel mask to be used for network operation.
CS_CHANNEL_PAGESpecifies a channel page to be used.
CS_EXT_PANIDSpecifies the network extended PANID. Should be unique at a given location.
CS_UIDThe extended address for platforms where it is not available via the special UID chip or an external EEPROM.

The description of parameters common for all applications can be found in Sample Applications Overview. ConfigServer parameters are described in the Configuration Server section.
