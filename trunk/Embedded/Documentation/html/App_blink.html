Blink Application

This example introduces the simplest application that uses the timer and LEDs if present on the board. When started, the application makes LEDs on the target board blink synchronously with a certain period.

The frequency of blinking is varied automatically every ten seconds, decreasing from the default APP_BLINK_PERIOD value by a factor of two and then increasing back to the default value.

In addition, if the board has buttons on it, the frequency of blinking may be varied manually within the range specified at compile time. Namely, it increases or decreases by a factor of two on pressing on the first or the second button (on the MeshBean board, SW1 and SW2 buttons), respectively. Minimum, maximum and initial period values (in milliseconds) are defined in the configuration.h file.

It is not recommended to set the initial period outside the minimum..maximum range, or set the minimum value to less than 2. In these cases, the application may behave inconsistently. (The corresponding error checking was omitted intentionally in order to keep the application code as simple as possible.)

Configuration parameters

APP_BLINK_PERIODInitial blinking period (ms)
APP_MIN_BLINK_PERIODMinimum blinking period (ms)
APP_MAX_BLINK_PERIODMaximum blinking period (ms)

The description of parameters common for all applications can be found in Sample Applications Overview. ConfigServer parameters are described in the Configuration Server section.

The source code and for the Blink application is located in the [SDK]\Sample Applications\Blink directory.
